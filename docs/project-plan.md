# Pinterest 개인용 다운로더 계획서

## 1. 프로젝트 개요

이 프로젝트는 Pinterest에서 참고 이미지를 더 빠르게 수집하기 위한 개인용 크롬 확장프로그램이다.

목표는 사용자가 다음 흐름을 한 번에 처리할 수 있게 하는 것이다.

1. Pinterest 핀 상세 페이지를 연다.
2. 현재 크게 보고 있는 메인 이미지를 인식한다.
3. 주변에 표시되는 유사 이미지들을 인식한다.
4. 원하는 이미지를 여러 장 선택한다.
5. 선택한 이미지를 `PNG` 형식으로 저장한다.

이 프로젝트는 크롬 웹스토어 배포용이 아니라, 개인이 로컬에서 설치해 사용하는 도구를 전제로 한다.

## 2. 제품 목표

이 확장프로그램은 다음과 같은 반복 작업을 줄이는 데 목적이 있다.

- 이미지를 하나씩 열어서 저장하는 작업
- 유사 이미지를 개별적으로 다시 찾는 작업
- 참고 이미지 수집에 드는 시간과 클릭 수

기대 효과는 다음과 같다.

- 한 화면에서 참고 이미지 여러 장을 빠르게 수집
- 메인 핀과 유사 이미지의 동시 선택
- 일관된 파일명과 형식으로 정리된 저장

## 3. 범위 정의

### 포함 범위

- Pinterest 상세 페이지에서 메인 핀 이미지 감지
- 메인 이미지 주변의 유사 이미지 카드 감지
- 감지된 이미지 위에 선택 UI 표시
- 다중 선택
- 일괄 다운로드
- 저장 형식: `PNG`
- 간단한 설정 UI 제공

### MVP 제외 범위

- 무한 스크롤 자동 수집
- Pinterest 비공개 API 의존
- 계정 동기화
- 클라우드 업로드
- OCR, 태깅, AI 분류
- 여러 페이지를 넘나드는 중복 관리
- 모바일 브라우저 대응

## 4. 핵심 사용자 흐름

1. 사용자가 Pinterest 핀 상세 페이지를 연다.
2. 확장프로그램 아이콘 또는 화면 내 버튼으로 선택 모드를 시작한다.
3. 확장프로그램이 다음 요소를 하이라이트한다.
   - 현재 보고 있는 메인 핀 이미지
   - 주변에 노출된 유사 이미지 카드
4. 사용자가 원하는 이미지를 클릭하여 선택 또는 해제한다.
5. 다운로드를 실행한다.
6. 다운로드 버튼을 누른다.
7. 선택한 이미지들이 로컬로 저장된다.

## 5. 기능 요구사항

### FR-01. Pinterest 페이지 감지

- 확장프로그램은 Pinterest 페이지에서만 동작해야 한다.
- 현재 페이지가 메인 핀이 보이는 상세 화면인지 판별해야 한다.

### FR-02. 메인 이미지 감지

- 현재 보고 있는 대표 핀 이미지를 찾아야 한다.
- 메인 이미지는 다른 이미지와 구분되게 표시되어야 한다.

### FR-03. 유사 이미지 감지

- 메인 이미지 주변에 렌더링된 유사 이미지 카드를 식별해야 한다.
- MVP에서는 현재 DOM에 보이는 항목만 지원한다.

### FR-04. 다중 선택

- 사용자는 여러 이미지를 동시에 선택하거나 해제할 수 있어야 한다.
- 선택된 상태는 화면에서 직관적으로 보여야 한다.

### FR-05. 일괄 다운로드

- 선택한 모든 이미지를 한 번의 동작으로 다운로드할 수 있어야 한다.
- 현재 화면에서 감지된 항목 기준으로 안정적으로 동작해야 한다.

### FR-06. 출력 형식

- 다음 저장 흐름을 지원해야 한다.
  - 직접 저장 가능한 큰 이미지 다운로드
  - 필요 시 `PNG`로 변환 후 저장

### FR-07. 파일명 규칙

- 저장되는 파일명은 예측 가능해야 한다.
- 예시:
  - `pinterest-main-001.png`
  - `pinterest-similar-002.png`

### FR-08. 오류 처리

- 특정 이미지가 내려받기 또는 변환에 실패해도 어떤 항목이 실패했는지 표시해야 한다.
- 일부 실패가 있더라도 나머지 항목은 계속 처리되어야 한다.

## 6. 비기능 요구사항

### 성능

- 일반적인 상세 페이지에서 선택 오버레이가 1초 내로 표시되어야 한다.
- 최대 20장 수준의 선택 항목은 탭을 멈추게 하지 않고 다운로드를 시작해야 한다.
- 변환 처리는 순차 또는 제한된 큐 방식으로 수행해 메모리 급증을 막아야 한다.

### 안정성

- 일부 카드 파싱에 실패해도 전체 기능이 중단되지 않아야 한다.
- Pinterest DOM 구조가 일부 바뀌더라도 최소한 핵심 흐름은 유지되도록 설계해야 한다.

### 유지보수성

- Pinterest 전용 셀렉터는 한 모듈에 모아 관리해야 한다.
- 다운로드 로직과 화면 UI 로직은 분리해야 한다.

### 사용성

- 별도 설명 없이도 선택 상태, 다운로드 상태, 오류 상태를 구분할 수 있어야 한다.
- 메인 이미지와 유사 이미지의 구분이 시각적으로 명확해야 한다.

## 7. 권장 MVP 범위

### MVP 1차

- 확장 아이콘으로 수동 실행
- 메인 핀 이미지 감지
- 화면에 보이는 유사 이미지 감지
- 체크박스 형태의 다중 선택 UI
- 일괄 다운로드
- `PNG` 저장
- 간단한 진행 상태 표시

### 1.1 확장

- `현재 보이는 항목 전체 선택` 버튼
- 파일명 접두어 설정
- 선택 개수 표시 강화
- 실패 항목 재시도

### 1.2 확장

- Pinterest 재렌더링 시 오버레이 자동 재적용
- 중복 이미지 URL 제거
- 날짜나 키워드 기반의 저장 규칙 추가

## 8. 리스크 평가

### 리스크 1. Pinterest DOM 변경

Pinterest는 동적 SPA이므로 클래스명이나 구조가 바뀌면 감지 로직이 깨질 수 있다.

대응:

- 셀렉터를 한곳에 모아 관리
- 고정 클래스명보다 위치와 크기 기반 휴리스틱 활용
- 필요하면 수동 선택 보조 모드 추가

### 리스크 2. 이미지 접근 제한

일부 이미지 URL은 형식, 헤더, 접근 제한 때문에 직접 저장이 어려울 수 있다.

대응:

- 우선 직접 다운로드 시도
- 실패 시 Blob으로 가져와 변환
- 그래도 실패하면 해당 항목만 오류로 표시

### 리스크 3. 대량 처리 시 메모리 부담

고해상도 이미지를 많이 선택하면 메모리 사용량이 커질 수 있다.

대응:

- 동시 처리 수 제한
- 큐 기반 다운로드와 변환 적용

## 9. 성공 기준

다음 조건을 만족하면 MVP 성공으로 본다.

- 일반적인 Pinterest 상세 페이지에서 메인 이미지가 안정적으로 감지된다.
- 한 화면에서 보이는 유사 이미지 6~12장 이상을 선택할 수 있다.
- 선택 이미지를 한 번에 저장할 수 있다.
- `PNG` 저장이 정상 동작한다.
- 일부 실패가 있어도 전체 작업이 중단되지 않는다.

## 10. 권장 개발 순서

1. 확장프로그램 기본 골격 구성
2. Pinterest 페이지 감지
3. 메인 이미지와 유사 이미지 탐지
4. 선택 오버레이 UI 구현
5. 다운로드 파이프라인 구현
6. 이미지 변환 처리
7. 진행 상태와 오류 UI 추가
8. 셀렉터 보강 및 예외 처리

## 11. 최종 권장 방향

이 프로젝트는 개인용 기준으로 충분히 실현 가능하며, MVP 방식으로 접근하는 것이 가장 효율적이다.

권장 방향은 다음과 같다.

- 핀 상세 페이지의 현재 화면 범위에 집중
- 숨은 API 대신 화면 DOM 기반으로 구현
- 셀렉터 수정이 쉬운 구조로 설계
- 자동화 범위를 넓히기보다 안정성을 우선
